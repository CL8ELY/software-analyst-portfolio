Esta sección describe los principales tipos de artefactos que forman parte del portafolio y el criterio utilizado para su publicación.

## Tipos de artefactos incluidos

Según el proyecto, el portafolio podrá incorporar documentos como:

- visión general del sistema,
- requisitos funcionales,
- requisitos no funcionales,
- reglas de negocio,
- casos de uso,
- historias de usuario,
- criterios de aceptación,
- glosario,
- matriz de trazabilidad,
- riesgos y supuestos,
- actas o resúmenes de reuniones,
- descripciones AS-IS y TO-BE,
- preguntas frecuentes funcionales y técnicas.

## Propósito de esta biblioteca

La biblioteca de artefactos existe para demostrar:

- capacidad de análisis y estructuración,
- calidad documental,
- consistencia editorial,
- y criterio para transformar documentación tradicional en un formato moderno y mantenible.

## Criterios de publicación

No todo documento histórico debe publicarse de la misma manera.  
Cada artefacto se evalúa bajo uno de estos criterios:

### 1. Publicable casi tal cual

Aplica cuando el contenido puede compartirse sin exponer información sensible y mantiene valor profesional por sí mismo.

### 2. Publicable con resumen

Aplica cuando el documento original es demasiado extenso, repetitivo o está orientado al uso interno, pero puede convertirse en una versión ejecutiva o estructurada para el portafolio.

### 3. Publicable con anonimización

Aplica cuando el contenido tiene valor analítico, pero contiene referencias a nombres internos, datos organizacionales, cifras privadas o información reservada que debe generalizarse o eliminarse.

### 4. No publicable

Aplica cuando la sensibilidad del contenido supera el beneficio de incluirlo públicamente.

## Principios editoriales

Los artefactos publicados en este portafolio siguen estos principios:

- fidelidad a la experiencia real,
- claridad estructural,
- lenguaje profesional,
- mínima ambigüedad,
- anonimización responsable,
- y utilidad para revisión funcional o técnica.

## Transformación desde documentación histórica

La mayor parte del contenido que alimentará esta biblioteca proviene de documentos corporativos elaborados originalmente en formatos tradicionales.  
La transformación hacia este portafolio implica:

- clasificar el documento fuente,
- identificar su propósito real,
- mapear su estructura a un formato moderno,
- eliminar redundancias,
- mejorar navegabilidad,
- y mantener el valor analítico original.
