Esta sección describe los principales tipos de artefactos que forman parte del portafolio y el criterio utilizado para su publicación. La intención no es mostrar una acumulación de documentos, sino una colección de piezas funcionales que evidencian análisis, estructura y trazabilidad.

## Tipos de artefactos incluidos

Según el proyecto, el portafolio podrá incorporar documentos como:

- visión general del sistema,
- requisitos funcionales,
- requisitos no funcionales,
- reglas de negocio,
- casos de uso,
- historias de usuario,
- criterios de aceptación,
- glosario,
- matriz de trazabilidad,
- riesgos y supuestos,
- actas o resúmenes de reuniones,
- descripciones AS-IS y TO-BE,
- preguntas frecuentes funcionales y técnicas.

## Propósito de esta biblioteca

La biblioteca de artefactos existe para demostrar:

- capacidad de análisis y estructuración,
- calidad documental,
- consistencia editorial,
- y criterio para transformar documentación tradicional en un formato moderno y mantenible.

## Criterios de publicación

No todo documento histórico debe publicarse de la misma manera. Cada artefacto se evalúa bajo uno de estos criterios:

### Publicable casi tal cual

Aplica cuando el contenido puede compartirse sin exponer información sensible y mantiene valor profesional por sí mismo.

### Publicable con resumen

Aplica cuando el documento original es demasiado extenso, repetitivo o está orientado al uso interno, pero puede convertirse en una versión ejecutiva o estructurada para el portafolio.

### Publicable con anonimización

Aplica cuando el contenido tiene valor analítico, pero contiene referencias a nombres internos, datos organizacionales, cifras privadas o información reservada que debe generalizarse o eliminarse.

### No publicable

Aplica cuando la sensibilidad del contenido supera el beneficio de incluirlo públicamente.
